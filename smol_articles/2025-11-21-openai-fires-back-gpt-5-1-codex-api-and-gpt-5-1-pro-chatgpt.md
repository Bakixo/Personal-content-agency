Bu modelin, hepimizin günlük hayatta kullandığı ChatGPT'nin yeni beyni olması bekleniyor. "Pro" eklentisi, mevcut modellere göre çok daha gelişmiş yetenekler vadediyor. Peki neler olabilir?

*   **Daha Üstün Akıl Yürütme:** Karmaşık, çok adımlı problemleri daha tutarlı ve mantıklı bir şekilde çözebilme yeteneği. Örneğin, bir istatistik problemini sadece çözmekle kalmayıp, çözüm adımlarını teorik altyapısıyla birlikte açıklayabilmesi.

*   **Hız ve Verimlilik:** Daha hızlı cevap süreleri ve daha az kaynak tüketimi. Bu, özellikle anlık analiz ve interaktif uygulamalar için kritik bir öneme sahip.

### Basit Bir Kod Örneği: Gelecek Nasıl Olabilir?

Hayali GPT-5.1 Codex'e şöyle bir komut verdiğimizi düşünelim:

**Prompt (İstem):**
`"Bir Pandas DataFrame'im var. 'cihaz_verileri' adında bir CSV dosyasından geliyor. Bu veride 'sicaklik' ve 'nem' sütunları bulunuyor. Bu iki sütundaki aykırı değerleri (outliers) Z-skoru yöntemine göre tespit eden ve bu aykırı değerleri, ait oldukları sütunun medyan değeri ile değiştiren bir Python fonksiyonu yaz. Fonksiyon, işlem sonrası kaç tane aykırı değerin değiştirildiğini de raporlasın."`

**GPT-5.1 Codex'in Muhtemel Cevabı:**

```python
import pandas as pd
import numpy as np

def clean_outliers_with_zscore(df: pd.DataFrame, columns: list, threshold: float = 3.0) -> (pd.DataFrame, dict):
    """
    Verilen DataFrame sütunlarındaki aykırı değerleri Z-skoru yöntemine göre bulur
    ve bu değerleri sütunun medyanı ile değiştirir.

    Args:
        df (pd.DataFrame): İşlem yapılacak DataFrame.
        columns (list): Aykırı değerlerin aranacağı sütun adları.
        threshold (float): Aykırı değer olarak kabul edilecek Z-skoru eşiği.

    Returns:
        pd.DataFrame: Aykırı değerleri temizlenmiş DataFrame.
        dict: Her sütun için kaç adet değerin değiştirildiğini içeren rapor.
    """
    df_cleaned = df.copy()
    report = {}

    for col in columns:
        if col not in df_cleaned.columns:
            print(f"Uyarı: '{col}' sütunu DataFrame'de bulunamadı.")
            continue

        # Sütunun sayısal olup olmadığını kontrol et
        if not pd.api.types.is_numeric_dtype(df_cleaned[col]):
            print(f"Uyarı: '{col}' sütunu sayısal olmadığı için işlem atlandı.")
            continue
            
        # Z-skorlarını hesapla
        z_scores = np.abs((df_cleaned[col] - df_cleaned[col].mean()) / df_cleaned[col].std())
        
        # Aykırı değerlerin indekslerini bul
        outlier_indices = z_scores > threshold
        num_outliers = outlier_indices.sum()
        
        if num_outliers > 0:
            # Medyan değerini hesapla
            median_val = df_cleaned[col].median()
            
            # Aykırı değerleri medyan ile değiştir
            df_cleaned.loc[outlier_indices, col] = median_val
            
        report[col] = num_outliers
        
    return df_cleaned, report

# --- Örnek Kullanım ---
# Örnek bir veri oluşturalım
data = {
    'sicaklik': [22, 23, 21, 25, 24, 50, 22, 26, 23, -10],
    'nem': [60, 65, 62, 70, 68, 66, 95, 63, 67, 59]
}
cihaz_verileri_df = pd.DataFrame(data)

# Fonksiyonu çağıralım
temiz_df, degisiklik_raporu = clean_outliers_with_zscore(
    cihaz_verileri_df, 
    columns=['sicaklik', 'nem']
)

print("Değişiklik Raporu:")
print(degisiklik_raporu)

print("\nTemizlenmiş DataFrame:")
print(temiz_df)

```
